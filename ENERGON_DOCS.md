# Megatron-Energon Integration Guide

This document explains how this project integrates with `megatron-energon` for multimodal data loading, focusing on the use of `InterleavedSample`, `Cookers`, and `TaskEncoder` for complex VLM training.

## Data Schema & Loading

The synthetic datasets generated by `src/generate.py` are saved as standard `WebDataset` shards (`.tar` files) containing:
- `json`: Metadata with conversational turns.
- `jpg`, `jpg_1`, ...: Multiple images associated with the sample.

### CrudeSample vs. InterleavedSample
When using `energon prepare`, we typically use `--sample-type CrudeSample` to maintain flexibility. Energon's `CrudeSample` is a raw dictionary-like object representing a single entry in the WebDataset.

To support complex multimodal inputs (interleaved images and text), we transform these into `InterleavedSample` objects using **Cookers**.

## Cookers: Transforming Raw Data
Cookers are stateless functions that preprocess a raw sample before it reaches the `TaskEncoder`. In `src/task_encoders.py`, `cooker_captioning` handles:
- Mapping roles (`human` to `user`, `gpt` to `assistant`).
- Extracting images from the shard and pairing them with conversation turns. Note that this images are given as `torch.Tensor` by Energon.
- Creating a `sequence` of items (images and messages) within an `InterleavedSample`.

```python
@stateless
def cooker_captioning(sample: dict) -> InterleavedSample:
    # Transforms CrudeSample into InterleavedSample
    ...
    return InterleavedSample(
        **basic_sample_keys(sample),
        sequence=sequence,
    )
```

## TaskEncoder: Tokenization and Batching
The `TaskEncoder` is responsible for converting the high-level `InterleavedSample` into tensors suitable for model training.

### 1. `encode_sample`
Each sample is tokenized using the model's processor (e.g., `Qwen3-VL`). This step handles:
- Applying chat templates.
- Resizing and normalizing images.
- Mapping image tokens to vision-specific IDs.

### 2. `batch` (Data Packing)
Instead of simple padding, this project demonstrates **Data Packing**. Multiple variable-length samples are concatenated into a single long sequence (up to `max_length`).
- `input_ids`: Concatenated token IDs.
- `attention_mask`: Masking for padded areas (if any).
- `cu_seqlens`: Cumulative sequence lengths used to separate different samples in the same batch, enabling Flash Attention and other optimizations.

## Visualization & Inspection
Understanding how tokens are distributed across the batch is critical for debugging packing efficiency.

### Token Mapping (`src/viz_synthetic.py`)
Visualizes the token distribution by categorizing IDs into:
- **Padding**: Empty space at the end of the batch.
- **User/Assistant Text**: Standard language tokens.
- **Image Tokens**: Tokens reserved for vision features.
- **Special Tokens**: BOS, EOS, and role markers.
- **Boundary**: Sequence boundaries identified by `cu_seqlens`.

### Text Decodin (`src/viz_text.py`)
Decodes the `input_ids` and prints them, respecting `cu_seqlens` to show how samples are packed.

## Typical Workflow
1. **Prepare**: `energon prepare data/path --sample-type CrudeSample`
2. **Train/Visualize**: Use `get_train_dataset` with your custom `TaskEncoder` which contains the appropriate `Cookers`.
